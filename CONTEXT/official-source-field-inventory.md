# Official Source Field Inventory

Status: E1 inventory for AC15-AC16.

Scope: KBO official sources currently represented in backend code, committed fixtures, and tests. No non-official, third-party, Naver, or Daum field adoption is permitted by this matrix.

## Endpoint Observations

- `GetKboGameDate`: official POST `Main.asmx/GetKboGameDate`; validates date navigation keys `BEFORE_G_DT`, `NOW_G_DT`, `NOW_G_DT_TEXT`, `AFTER_G_DT`, `code`, and `msg`. It selects the KBO date but does not directly map game display fields.
- `GetKboGameList`: official POST `Main.asmx/GetKboGameList`; primary source for `G_ID`, team ids/names, status codes, score, live inning/count/base/current fields, starter names/ids, and score-derived box score runs. Committed completed and live raw dumps contain this payload.
- `GetScheduleList`: official POST `Schedule.asmx/GetScheduleList`; source for schedule-only games, start time, venue, broadcast channels, game-center links, and cancelled/delayed notes. It may infer a `gameId` from official schedule row date and team names when no link is present.
- `LiveTextView2`: official POST `Game/LiveTextView2.aspx`; source for previous at-bat text parsed from KBO HTML lines such as `normaiflTxt`. It enriches only requested-date live games.
- `TeamRankDaily`: official HTML `Record/TeamRank/TeamRankDaily.aspx?date=YYYYMMDD`; source for team rank, wins, losses, draws, streak, win rate, recent ten, and games back. Current normalized `teamRecords` adopts only wins/losses/draws/rank/streak.
- `GetPitcherRecordAnalysis`: official POST `Schedule.asmx/GetPitcherRecordAnalysis`; source for scheduled starter season record snippets by pitcher cell and side-specific stat classes. Current route enrichment reads stored pitcher season summaries populated by official collection flows.
- Official collection/dump scripts: `backend-spike/scripts/dump-kbo-response.ts` writes KBO game date/game list/schedule dumps; `backend-spike/scripts/poll-live-games.ts` captures live raw/normalized snapshots and now persists `changes/*.raw.json` when `--save-raw` and change capture are enabled; `backend-spike/scripts/collect-source-data.ts` writes official date, standings, player, and extra-record artifacts; `backend-spike/scripts/collect-probable-pitcher-records.ts` collects starter record analysis into DB-backed pitching summaries. Artifact outputs under `backend-spike/artifacts/` are collection products, not committed fixture evidence unless promoted.
